## ccmix
`ccmix` - это инструмент командной строки для создания и извлечения из архива файлов в формате Westwood Studios `.mix`. 
Инструмент может создать любой из трех основных вариантов, использовавшихся в классических играх Westwood. 
Это версии Tiberian Dawn, Red Alert с зашифрованными и незашифрованными заголовками файлов и Tiberian Sun с зашифрованными и незашифрованными заголовками файлов. 
Кроме того, в расширении XCC предусмотрена возможность включения локальной базы данных имен файлов для восстановления имен файлов из одностороннего хэша, в котором они хранятся.

Использование  
Обычно используется команда `ccmix --mode --mix /path/to/file.mix` с различными опциями, которые изменяют поведение программы при создании или извлечении файлов. В настоящее время поддерживаются следующие режимы:  
`--extract`					Извлечет файл или все файлы из микса. Требуется опция `--mix`.  
`--create`					Создаст файл микса из содержимого каталога. В настоящее время требует минимум опций `--dir` и `--mix`.  
`--list`					Перечисляет содержимое микс-файла. Требуется опция `--mix`.  
`--add`						Попытается перестроить микс-файл и добавить запрашиваемый файл.  
`--delete`					Это приведет к попытке перестроить микс-файл и удалить запрошенный файл.  

Также могут быть использованы следующие опции:  
`--file /path/to/file.name`	Указывает конкретный файл для извлечения.  
`--dir /path/to/dir`		Указывает папку, в которую нужно извлечь файл, или содержимое папки, из которой нужно сделать .mix.  
`--game`					Может быть td, ra, ts или ra2. Указывает, для какой игры предназначен обрабатываемый микс. td охватывает Sole Survivor, а также C&C, а ra2 - Yuri's Revenge. Если не указано, ccmix по умолчанию использует `td`.  
`--encrypt`					Используется при создании файла, шифрует заголовок с помощью blowfish. Ключ частично генерируется из rand(). Требуется `--game ra` или более поздняя версия.  
`--checksum`				Используется для включения контрольной суммы SHA1, если она установлена, игра будет использовать ее для проверки содержимого файла mix и откажется загружать, если она не совпадает. Требуется `--game ra` или более поздняя версия.  
`--lmd`						Указывает, следует ли генерировать и включать в микс локальный файл database.dat, соответствующий формату XCC.

## CSFTool
CSFTool - инструмент командной строки для преобразования текстовых файлов в файлы строковых таблиц (`.csf`), используемых в играх Command & Conquer от Westwood Studios, и наоборот (в `.txt`).

Принимаемые параметры:  
`-h`, `-?`, `--help` 				Показать справку.  
`-i`, `--infile=VALUE` 				Входной файл строковой таблицы.  
`-o`, `--outfile=VALUE` 			Выходной файл строковой таблицы.  
`-t`, `--textfile=VALUE` 			Имя входного/выходного текстового файла. Если не указано, по умолчанию используется имя входной строковой таблицы с расширением `.txt`.  
`-a`, `--addlines`					Добавлять строки из текстового файла в таблицу строк в виде строк. Задание этого параметра устанавливает значение `-e` в `false`.  
`-e`, `--exportlines` 				Экспорт строк из таблицы строк в строки текстового файла.  
`-l`, `--language-override=VALUE` 	Задайте целое число для переопределения сохраненного идентификатора языка таблицы строк. Допустимые значения от 0 до 9 и специальное значение -1 (независимость от языка).  
`-d`, `--debug-logging` 			Если установлено, журнал записывается в файл в каталоге программы.  

Текстовые файлы для преобразования в строковую таблицу должны содержать строковые метки и значения в формате `LABEL|VALUE`, каждое в отдельной строке. Символы `\n` являются переходом на новую строку.

## BagFileTool
BagFileTool - программа командной строки для создания и изменения `.idx`, `.bag` файлов из игр Westwood Studios, а также извлечения их содержимого.

Принимаемые параметры:  
`-h`, `-?`, `--help`                Показать справку.  
`-i`, `--input-filename=VALUE`    	Входной файл.  
`-o`, `--output-filename=VALUE`   	Выходной файл.  
`-a`, `--add-files=VALUE`         	Список файлов (и/или каталогов - все содержащиеся в них файлы будут добавлены рекурсивно), разделенных запятыми, для добавления в файл `.bag`.  
`-e`, `--extract-files=VALUE`     	Список имен файлов (без расширения), которые нужно извлечь из файла `.bag`, разделенный запятыми. Если список пуст или содержит подстановочный знак (\*), будут извлечены все файлы.  
`-l`, `--log`                     	Если установлено, журнал записывается в файл в каталоге программы.  
`-d`, `--debug`                   	Если установлено, в консольном окне отображается журнал уровня отладки.  
`-x`, `--auto-index`              	Если установлено, автоматически генерирует выходные файлы `.bag` и `.idx` в виде audioXX.bag и audioXX.idx, где XX - это число от 00 до 99.   
`-n`, `--index-number=VALUE`      	По аналогии с предыдущим параметром, только можно точно задавать индексный номер (00-99) для генерации имен файлов audioXX.bag и audioXX.idx.   
`-y`, `--yes`                     	Если установлено, автоматически перезаписывает существующие файлы без запроса.

## Inifile
inifile - инструмент командной строки для работы с INI-файлами (под Windows) с именами секций в квадратных скобках и назначениями вида: `item`=`string`.  

Синтаксис:  
`INIFILE filename [section] item=string` 		Изменить или перезаписать элемент  
`INIFILE filename [section] item=`				Удалить элемент  
`INIFILE filename [section] item==` 			Очистить элемент  
`INIFILE filename [section] /remove` 			Удалить всю секцию  
`INIFILE filename [section] item > temp.bat` 	Прочитать элемент  
`INIFILE filename [section] > temp.bat` 		Прочитать все элементы раздела  

Элементы и секции генерируются, если не найдены. Файл INI должен существовать.  
Чтение задания отправляет оператор `SET` в `STDOUT`, который может быть перенаправлен во временный пакетный файл или обработан с помощью конструкции `FOR /F`.

## xxd
xxd - утилита типа hex-dump.

Использование: `xxd [options] [Infile [Outfile]]`  
или `xxd -r [-s [-] offset] [-c cols] [-ps] [Infile [Outfile]]`

Параметры:  
`-a` 			Переключатель AutoSkip: один `*` заменяет NUL-Lines. По умолчанию выключен.  
`-b`			Двоичный цифровой дамп (несовместимый с `-ps`, `-i`, `-r`). По умолчанию hex.  
`-c cols` 		Формат <Cols> октеты на строку. По умолчанию 16 (`-i: 12`, `-ps: 30`).  
`-E` 			Показать символы в EBCDIC. По умолчанию ASCII.  
`-e` 			little-endian дамп (несовместима с `-ps`, `-i`, `-r`).  
`-g` 			Номер октета на группу в обычном выходе. По умолчанию 2 (`-e: 4`).  
`-h` 			Помощь.  
`-i` 			Вывод в файл в C-стиле.  
`-l len` 		Остановить после `<len>` октета.  
`-o off` 		Добавить `<off>` в отображаемое положение файла.  
`-ps` 			Вывод в PostScript-стиле (непрерывный формат вывода шестнадцатеричного кода).  
`-r` 			Обратная операция: преобразовать (или изменение) шестнадцатеричного дампа в двоичный.  
`-r -s off` 	Возвращает с `<off>` добавленным в позиции файлов, найденные в шестнадцатеричном дампе.   
`-s [+][-]seek`	Работа с указанного абсолютного `<seek>` (или относительного) смещения в байтах.  
`-u` 			Использовать верхний регистр в шестнадцатеричном представлении.  
`-v`			Показать версию.

## BinkMix
BinkMix - CLI-утилита от устаревшей версии программы RAD Tools (1.0), для пакетной обработки файлов Bink.  

Синтаксис: `BinkMix <Входной файл Bink>[.BIK] <Входной звуковой файл> <Выходной файл>[.BIK] [/Доп.Опции]`  
`/L#` 		Уровень сжатия звука: 0=без потерь, 4=почти без потерь, 99=с очень большими потерями.  
`/T#` 		Идентификатор дорожки, в которую нужно поместить звук (по умолчанию 0).  
`/N#` 		Указывает начальный кадр (отрицательное значение для смещения в миллисекундах).  
`/P` 		Зациклить звук, если звук короче файла Bink.  
`/F#` 		Переопределить частоту кадров вывода.  
`/R#` 		Принудительно установить скорость выходных данных (11025, 22050, 44100 и т.д. Hz).  
`/B#` 		Принудительно выдать выходные биты (8 или 16).  
`/C#` 		Принудительно изменить выходной канал (1=моно, 2=стерео).  
`/I#` 		Номер звуковой дорожки, с которой считывается звук.  
`/S#` 		Указать время начала в миллисекундах.  
`/E#` 		Указать время окончания в миллисекундах.  
`/Y#` 		Принудительно установить тип входного файла: 1=8 моно, 2=8 стерео, 3=16 моно, 4=16 стерео.  
`/D#` 		Указать принудительную скорость передачи данных входного файла (используйте с переключателем `/Y`).  
`/K#`		Указать, сколько байт нужно пропустить во входной файл (используйте с `/Y`).  
`/O`		Автоматически перезаписывать выходной файл.

## FFMpeg
ffmpeg - это бесплатный и открытый набор библиотек и утилит для обработки мультимедиа. 

Синтаксис: `ffmpeg [опции] [[опции входного файла] -i входной_файл]... {[опции выходного файла] выходной_файл}...`

Получение справки:  
`-h`      											Показать основные опции.  
`-h long`											Показать больше опций.  
`-h full`											Показать все опции (включая все специфичные для форматов и кодеков, очень длинно).  
`-h type=name`										Показать все опции для указанного декодера/кодера/демультиплексора/мультиплексора/фильтра/bsf/протокола.  

Опции для отдельных потоков могут сопровождаться `:<спецификатор_потока>` для применения только к указанным потокам. `<спецификатор_потока>` может быть индексом потока или `v/a/s` для видео/аудио/субтитров (полный синтаксис см. в руководстве).  

Показать справку / информацию / возможности:  
`-L`                  								Показать лицензию.  
`-h <тема>`           								Показать справку.  
`-version`            								Показать версию.  
`-muxers`             								Показать доступные мультиплексоры (muxers).  
`-demuxers`           								Показать доступные демультиплексоры (demuxers).  
`-devices`            								Показать доступные устройства.  
`-decoders`           								Показать доступные декодеры.  
`-encoders`           								Показать доступные кодеры.  
`-filters`            								Показать доступные фильтры.  
`-pix_fmts`           								Показать доступные форматы пикселей.  
`-layouts`            								Показать стандартные раскладки каналов.  
`-sample_fmts`        								Показать доступные форматы аудиосэмплов.  

Глобальные опции (влияют на всю программу, а не на один файл):  
`-v <уровень_логирования>` 							Установить уровень детализации логов.  
`-y`                  								Перезаписывать выходные файлы.  
`-n`                  								Никогда не перезаписывать выходные файлы.  
`-stats`              								Выводить отчет о прогрессе во время кодирования.  

Опции для файлов (входных и выходных):  
`-f <формат>`         								Принудительно задать формат контейнера (иначе определяется автоматически).  
`-t <длительность>`   								Остановить транскодирование после указанной длительности.  
`-to <время_окончания>` 							Остановить транскодирование по достижении указанного времени.  
`-ss <время_сдвига>`  								Начать транскодирование с указанного времени.  

Опции для файлов (только выходные):  
`-metadata[:<спецификатор>] <ключ=значение>` 		Добавить метаданные.  

Опции для потоков:  
`-c[:<спецификатор_потока>] <кодек>`  				Выбрать кодер/декодер (`copy` для копирования потока без перекодирования).  
`-filter[:<спецификатор_потока>] <граф_фильтров>`  	Применить указанные фильтры к аудио/видео.  

Видео опции:  
`-r[:<спецификатор_потока>] <частота>` 				Переопределить частоту кадров входного файла/преобразовать в заданную выходную частоту кадров (значение в Гц, дробь или аббревиатура).  
`-aspect[:<спецификатор_потока>] <соотношение>`  	Установить соотношение сторон (4:3, 16:9 или 1.3333, 1.7777).  
`-vn`                 								Отключить видео.  
`-vcodec <кодек>`     								Псевдоним для `-c:v` (выбор кодера/декодера для видеопотоков).  
`-vf <граф_фильтров>` 								Псевдоним для `-filter:v` (применить фильтры к видеопотокам).  
`-b <битрейт>`        								Видеобитрейт (пожалуйста, используйте `-b:v`).  

Аудио опции:  
`-aq <качество>`      								Установить качество аудио (зависит от кодека).  
`-ar[:<спецификатор_потока>] <частота>`  			Установить частоту дискретизации аудио (в Гц).  
`-ac[:<спецификатор_потока>] <каналы>`  			Установить количество аудиоканалов.  
`-an`                 								Отключить аудио.  
`-acodec <кодек>`     								Псевдоним для `-c:a` (выбор кодера/декодера для аудиопотоков).  
`-ab <битрейт>`       								Псевдоним для `-b:a` (выбор битрейта для аудиопотоков).  
`-af <граф_фильтров>` 								Псевдоним для `-filter:a` (применить фильтры к аудиопотокам).  

Опции для субтитров:  
`-sn`                 								Отключить субтитры.  
`-scodec <кодек>`     								Псевдоним для `-c:s` (выбор кодера/декодера для потоков субтитров).
